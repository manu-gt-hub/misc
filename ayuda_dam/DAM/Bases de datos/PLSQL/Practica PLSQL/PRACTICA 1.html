Resolver los siguientes apartados:

 

a)       Crear un bloque anónimo que permita insertar un registro en la tabla EMPLEADO. Antes de insertar el registro deben realizarse la siguientes validaciones:

·         Comprobar que el NIF tiene entre 7 y 8 números y una sola letra. En caso contrario insertar un error en la tabla INFORME.

·         Si no ha habido ningún error anteriormente, Pasar la letra del NIF a mayúsculas.

·         Si no ha habido ningún error anteriormente, quitar todos los ceros por la izquierda.

·         Si no ha habido ningún error anteriormente, validar la letra del NIF con el algoritmo siguiente. En caso de error se reflejará en la tabla INFORME.

·         Si no ha habido ningún error anteriormente, comprobar que el NIF no exista en la tabla EMPLEADO, en cuyo caso se insertará. En caso contrario se reflejará en la tabla INFORME.

 

 

                Cuando se ejecuta el bloque anónimo, solo puede aparecer uno de los siguientes mensajes:

 

1)       El registro se ha insertado en la tabla EMPLEADO.

2)       El bloque se ejecutó con errores consulte la tabla INFORME.

 

                NOTA IMPORTANTE:

1)       Todos los campos que se insertan en EMPLEADO deben estar en mayúsculas.

2)       Los registros en la tabla INFORME, se insertan con un IDE Autonumérico, lo cual indica que hay que hacer uso de un secuencial para esa tabla.

 

 

 

b)       Crear un bloque anónimo que permita actualizar un registro de EMPLEADO. La forma de realizar la actualización se ajustará a la siguiente funcionalidad:

1)       Se debe buscar si el registro existe. Si no existe se reflejará el error en la tabla INFORME y el proceso termina informando: “El bloque se ejecutó con errores consulte la tabla INFORME”.

2)       Si el registro existe, se pasan todos los campos a mayúsculas y se actualiza la tabla EMPLEADO. 

 

 

 

c)        Crear un bloque anónimo que permita eliminar un registro de EMPLEADO. La forma de realizar el borrado se ajustará a la siguiente funcionalidad:

1)       Se debe buscar si el registro existe. Si no existe se reflejará el error en la tabla INFORME y el proceso termina informando: “El bloque se ejecutó con errores consulte la tabla INFORME”.

2)       Si el registro existe, se elimina de la tabla EMPLEADO.

 

 

 

d)       Mostrar información de un registro de la tabla EMPLEADO. La forma de realizar esta operación es la siguiente:

1)       Se debe buscar si el registro existe. Si no existe se reflejará el error en la tabla INFORME y el proceso termina informando: “El bloque se ejecutó con errores consulte la tabla INFORME”.

2)       Si el registro existe, se debe mostrar la información con el siguiente formato:
