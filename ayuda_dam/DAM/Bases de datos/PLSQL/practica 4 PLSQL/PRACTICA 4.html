Dado el modelo Relacional

 

 

NOTA: Para realizar esta práctica debe crearse un paquete “pckFUN1”, “pckFUN2” y “pckFUN3” para cada uno de los ejercicios, de modo que dentro de cada paquete deben crearse todos los tipos de datos necesarios, funciones y procedimientos que permitan resolver el ejercicio.

 

Ejercicio 1: Cada cierto tiempo se recibe un listado de personal en la tabla PERSONAL. Dicha tabla esta en el modelo sin relacionar con ninguna otra tabla. Si la tabla está vacía, no se realiza ninguna actividad. Si la tabla está llena se realizan las siguientes operaciones:

               

-          Cada fila de la tabla contiene un IDE y un CONTENIDO.

-          IDE es un campo Autonumérico, su valor no tiene importancia en nuestro subsistema.

-          CONTENIDO es un campo de texto que contiene la siguiente información:

 

DNI#NOMBRE#APELLIDO1#APELLIDO2#DIRECCION#TELEFONO#EDAD#SUELDO#COMISION#

 

Ejemplos

11111111#PEDRO#ALONSO#GARCIA#C/ LAS MOSCAS 33#111111111#1000#5#

22222222#JOSE#RECIO#LORCA#C/ OSOS 15#222222222#1200#7#

33333333#NULL#NULL#NULL#NULL#NULL#NULL#NULL#

44444444#LOLA#ROMERO#LOPEZ#C/ LAS ABEJAS 76#333333333#900#3#

 

-          Debe crearse un cursor explicito que lea el campo contenido y lo descomponga en todos sus campos. Después debe aplicarse el siguiente algoritmo:

 

1.        Si el registro existe en COMERCIAL y el nombre es NULL se marca una baja lógica en COMERCIAL.

2.        Si el registro existe en COMERCIAL y el nombre no es NULL se actualiza el sueldo y la comisión en COMERCIAL.

3.        Si el registro no existe en COMERCIAL se da de alta el registro en COMERCIAL.

4.        Una vez que se procesa la tabla PERSONAL, deben eliminarse todos sus registros.

 

Ejercicio 2: Cada cierto tiempo se ejecuta un proceso que permite calcular las comisiones de los comerciales de la siguiente forma:

 

-          En primer lugar debemos hacer un tipo cursor que permita obtener aquellos comerciales con vehículos vendidos sin procesar.

-          Para cada comercial debemos insertar en la tabla COMISION un registro por vehículo vendido, en el que se indicará el precio del vehículo, el % de comisión y la comisión obtenida.

-          Una vez calculada la comisión debe marcarse el vehículo como procesado.

-          En caso de no existir vehículos sin procesar, el proceso terminará y no hará nada.

 

 

Ejercicio 3: En el sistema también existe una tabla que se llama REMESA. Al igual que la tabla PERSONAL no está relacionada en el modelo. Sin embargo tiene la siguiente función.

 

-          Si la tabla está vacía el sistema no realiza ninguna acción.

-          Si la tabla tiene registros debe realizar las siguientes acciones:

1.        Debe descomponer los datos de cada registro e insertarlos en las tablas correspondientes.

La descripción de cada registro es:

 

Marca#Matricula#Detalle#Precio#

 

Ejemplos

Citroen#1234BNJ#C8 – Azul – 2.5 Diesel#20000#

Renault#4321ABC#Clio – Rojo – 1.5 Gasolina#7000#

 

2.        Si la MARCA existe no se crea, si no existe se crea.

3.        Los vehículos son vehículos nuevos para poder ser vendidos, así que deben ser insertados en la tabla vehículos.

               Debe tenerse en cuenta que estos vehículos no estarán asignados a ningún cliente ni a ningún comercial, pero si que deben                pertenecer a una determinada marca.
