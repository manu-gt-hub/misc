Dado el siguiente modelo relacional.

 

 

 

Tenemos en la tabla ROL los nombres de los roles a los que pertenece un usuario. Un usuario tiene un único “rol” aunque cada rol puede pertenecer a varios usuarios.  Cada Rol tiene definidos permisos sobre los objetos de una aplicación. Se consideran objetos todos los componentes que componen cada una de las ventanas de la aplicación.

 

Se desean realizar en la base las siguientes acciones:

a)    Crear un paquete llamado pckUsuario que debe contener los siguientes objetos:

-          Función ExisteUSR (nomUSR).

Devuelve true si el usuario existe y false en caso contrario.

 

-          Función ClaveUSR (nomUSR).

Si el usuario existe, devuelve el IDE. En caso contrario devuelve 0.

 

-          Función CreaUSR (nomUSR,conUSR,mailUSR).

Si el usuario no existe lo da de alta en la base de datos (Usuario,Contraseña,mail) y devuelve true.

Si el usuario existe devuelve false.

 

-          Función EliminaUSR (nomUSR).

Si el usuario existe se elimina y devuelve true.

Si el usuario no existe devuelve false.

 

-          Función CambiaConUSR (nomUSR,oldConUSR,newConUSR).

Si el usuario existe y contraseña antigua correcta, se cambia la contraseña y devuelve 1.

Si el usuario existe y contraseña antigua incorrecta, no se cambia la contraseña y devuelve 2.

Si el usuario no existe devuelve 3.

 

b)   Crear un paquete llamado pckRolObj que debe contener los siguientes objetos:

-          Procedimiento ExisteIdeROL (nomROL,ideROL,existe)

Si el nombre de ROL existe, se devuelve el ide y existe se devuelve a true.

Si el nombre de ROL no existe, ide=0 y existe se devuelve a false.

 

-          Función CreaROL (nomROL).

Si el ROL existe devuelve false, si el ROL no existe se crea y devuelve true.

 

-          Procedimiento ExisteIdeOBJ (nomOBJ,ideOBJ,existe)

Si el nombre de OBJ existe, se devuelve el ide y existe se devuelve a true.

Si el nombre de OBJ no existe, ide=0 y existe se devuelve a false.

 

-          Función CreaOBJ (nomOBJ).

Si el OBJ existe devuelve false, si el OBJ no existe se crea y devuelve true.

 

-          Procedimiento AgregaRolObj (ideROL,ideOBJ,existe,agregado)

Se comprueba si existe la relación ideROL, ideOBJ en la tabla ROLOBJ:

                 Si existe se devuelve true en existe y se devuelve false en agregado.

                 Si no existe se devuelve false en existe y se devuelve true en agregado.

 

-          Procedimiento QuitaRolObj (ideROL,ideOBJ,existe,quitado)   

Se comprueba si existe la relación ideROL, ideOBJ en la tabla ROLOBJ:

                 Si existe se devuelve true en existe y se devuelve true en quitado.

                 Si no existe se devuelve false en existe y se devuelve false en quitado.

 

 

c)    Crear un bloque anónimo que haga uso de las funciones y procedimientos de los paquetes creados.

 

d)   Crear los siguientes disparadores.

 

1.       Al insertar en la tabla usuario se guardará un mensaje en la tabla CORREO para todos los administradores.

Dicho mensaje mostrará el nombre del nuevo usuario.

 

2.       Al cambiar la contraseña del usuario se guardará un mensaje en la tabla CORREO para ese usuario.

Dicho mensaje mostrará la contraseña antigua y la nueva.

 

3.       Al eliminar un usuario se guardará un mensaje en la tabla CORREO a ese usuario y a los administradores.

Dicho mensaje informará sobre la baja del usuario.
